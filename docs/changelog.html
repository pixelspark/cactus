Version 1.0.0
Initial Release.
Version 1.1.0
  • Fixed a bug in CMatrix::RotateMatrix that made rotation look very odd
  • Fixed bug in CVertex::Project
  • Made transparency (CColor::Transparency) work
    • Version 1.5.0
  • Fixed texturing bug (_u and _v weren't projected correctly)
  • Added lighting support (not fully functional though, phongshading looks weird when zooming in)
  • Added some asserts to the CMaterial getters and setters
  • Made some CMaterial members protected instead of public
  • Project settings, directory structure and configuration has been streamlined
  • Made Paul Nettle's Memory Manager work with a special debug version of the project
  • Rewritten CTriangle to hold CVertex* instead of CVertex to enable lighting (vertices now stored in CObject)
  • Fixed bugs in CColor methods Add and Scale
  • CObjectFactory rewritten to work with the new way of storing vertices (as explained above)
  • Disabled -QIfist (CTextureFactory generated ugly results because of this rounding mode)
  • Cleaned up cactus.h, removed some comments that were useless
  • Fixed bug in CLinkedList: Empty didn't do anything
    • Version 1.6.0
  • Memory Manager introduced (thanks to Paul Nettle) but not working very good
  • Phongshading fixed, works as expected now
  • Fixed some bugs in CObject; normalmatrix was scaled too in CObject::Scale
    • Version 1.6.5
  • Memory Manager was disabled again; GlowCode did not find any leaks
  • Performance improvements for CBuffer::Clear and CCanvas::Clear
  • Performance improvements for CCOlor routines Add, Multiply and Scale (using int-colors now)
  • Performance improvements for the CRasterizer
  • Disabled Transparency temporarily, bugging
  • Fixed bug in CObjectFactory, Tubes, Spirals and Torus Knots now work
  • Fixed several memory leaking CTextureFactory routines using CColor::MakeGradient
    • Version 1.6.6
  • Removed zbuffer negative flip for performance reasons
  • CMatrix::Transform is now inlined
  • Fixed INT_GETALPHA macro
  • Started developing ActiveX-control
    • Version 1.7.0
  • Fixed backface culling: invisible triangles were still drawn
  • Added FPU-tweak methods (fputweak.h) for better performance
  • Added function to write performance data to file (win32demo)
  • Cleaned up internal structure; class definitions are now in separate files
  • Inline-methods are now in one file (Inlines.h)
  • Version number now only resides in cactus.h
    • Version 1.8.0
  • Made cactus work with Visual Studio .NET 2003, older versions may be dropped in future versions
  • Fixed bug with Z-buffer not being filled correctly
    • Version 1.9.0
  • Added CSDLViewer in Sandstorm library for fast viewing
  • Cleaned up whole Sandstorm source code
  • Added support for identification-buffer
  • Added extra material mode for showing vertices as dots
  • Experimental anti-aliasing in CGDIViewer
  • Lots of small bug fixes and features
